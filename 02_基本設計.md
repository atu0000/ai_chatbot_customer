## データフロー

### ドキュメント登録フロー

```
1. ユーザーがファイルをアップロード
2. loader.py がファイル種別を判定し本文を抽出
3. テキストをチャンク（500文字、重複50文字）に分割
4. embedder.py が各チャンクを Embedding に変換
5. ChromaDB にベクトル＋メタデータ（ファイル名・ページ）を保存
6. UI にアップロード完了を表示
```

### 回答生成フロー

```
1. ユーザーが質問を入力・送信
2. 質問を Embedding に変換
3. ChromaDB で類似度上位 k 件（k=5）のチャンクを取得
4. チャンクと会話履歴をプロンプトに組み込む
5. Claude API に送信
6. 回答テキスト＋使用チャンク（出典）をUIに表示
```

## プロンプト設計（基本方針）

```
System:
  あなたは社内情報アシスタントです。
  以下の【参考資料】のみを根拠として質問に回答してください。
  資料に情報がない場合は「資料に該当情報がありません」と答えてください。
  回答は日本語で、簡潔に記述してください。

【参考資料】
{検索で取得したチャンクを列挙}

User:
{質問}
```

- 会話履歴は `messages` 配列に含め、マルチターンに対応
- ハルシネーション抑制のため「資料のみを根拠にする」を明示

## 状態管理（Streamlit Session State）

| キー | 型 | 内容 |
|------|----|------|
| `messages` | `list[dict]` | 会話履歴（role / content） |
| `sources` | `list[dict]` | 最新回答の出典チャンク |

---

## エラーハンドリング方針

| エラー種別 | 対処 |
|-----------|------|
| 未対応ファイル形式 | UI にエラーメッセージを表示、処理中断 |
| API キー未設定 | 起動時に `.env` を検証、未設定なら警告表示 |
| Claude API エラー | エラーメッセージをチャット欄に表示 |
| ドキュメント未登録で質問 | 「先にドキュメントをアップロードしてください」と返答 |

## 環境変数

| 変数名 | 内容 | 例 |
|--------|------|----|
| `ANTHROPIC_API_KEY` | Claude API キー | `sk-ant-...` |
| `CLAUDE_MODEL` | 使用モデル | `claude-sonnet-4-6` |
| `CHUNK_SIZE` | チャンク文字数 | `500` |
| `CHUNK_OVERLAP` | チャンク重複文字数 | `50` |
| `SEARCH_K` | 検索取得件数 | `5` |
