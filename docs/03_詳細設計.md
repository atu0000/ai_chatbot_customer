# 03 詳細設計

## rag/loader.py

### 責務
アップロードされたファイルを読み込み、チャンク分割した辞書リストを返す。

### 関数詳細

#### `load_document(file_path: str) -> list[dict]`

```
入力: ファイルパス（文字列）
出力: [{"text": str, "source": str, "page": int}, ...]
```

**処理フロー**

```
1. 拡張子を判定 (.pdf / .csv / .txt / .docx / .xlsx)
2. 対応する読み込み関数に振り分け
3. テキストをチャンク分割
4. 各チャンクに source（ファイル名）・page を付与して返却
```

**拡張子別処理**

| 拡張子 | ライブラリ | ページ情報 |
|--------|-----------|-----------|
| .pdf | PyMuPDF (`fitz`) | ページ番号（1始まり） |
| .csv | pandas | 行番号（1始まり） |
| .txt | 標準 io | 固定値 1 |
| .docx | python-docx | 段落番号（1始まり） |
| .xlsx | openpyxl | シート名＋行番号 |

**チャンク分割アルゴリズム**

```python
def _split_text(text: str, chunk_size: int, overlap: int) -> list[str]:
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunks.append(text[start:end])
        start += chunk_size - overlap
    return chunks
```

- `chunk_size` / `overlap` は環境変数から読み込む（デフォルト: 500 / 50）
- 空文字チャンクは除外する

**CSV の扱い**

- 各行を `列名: 値, 列名: 値, ...` 形式の文字列に変換してからチャンク分割
- ヘッダー行は本文には含めず、各行変換時に列名として使用する

**Word (.docx) の扱い**

- `python-docx` で全段落のテキストを順に取得する
- 段落番号を page に記録する（見出し・本文を区別しない）
- 表（Table）内のセルも走査してテキストを抽出する

**Excel (.xlsx) の扱い**

- `openpyxl` で全シートを走査する
- 各行を `列A: 値, 列B: 値, ...` 形式の文字列に変換する（CSV と同様）
- page には `"シート名_行番号"` 形式の文字列を設定する
- 空行はスキップする

## rag/embedder.py

### 責務
ChromaDB の操作（追加・検索・削除・一覧）を一元管理する。ユーザー個人コレクションと全体共有コレクションの両方を扱う。

### 定数・初期化

```python
CHROMA_PATH = "data/chroma"
EMBED_MODEL = "paraphrase-multilingual-mpnet-base-v2"
SHARED_USER = "__shared__"   # 共有コレクションを示す特別なユーザー名

_client = chromadb.PersistentClient(path=CHROMA_PATH)
_model = SentenceTransformer(EMBED_MODEL)
```

- コレクション名は `documents_{username}` の形式（例: `documents_alice`、`documents___shared__`）
- `PersistentClient` により再起動後もデータを保持する
- Embedding モデルはアプリ起動時に1度だけロードする

### 関数詳細

#### `add_documents(chunks: list[dict], username: str) -> None`

```
入力: [{"text": str, "source": str, "page": int}, ...], ユーザー名
処理:
  1. _get_collection(username) でコレクションを取得（なければ作成）
  2. 各チャンクの "text" を SentenceTransformer で Embedding に変換
  3. ID を MD5(source + page + text[:50]) で生成（重複登録を upsert で吸収）
  4. collection.upsert() でベクトル・本文・メタデータを保存
  5. キャッシュをクリア（list_sources, list_sources_with_count）
```

#### `search(query: str, username: str, k: int = 5) -> list[dict]`

```
入力: 質問文字列, ユーザー名, 取得件数
処理:
  1. query を Embedding に変換（1回のみ）
  2. 個人コレクション（username）と共有コレクション（__shared__）を両方 query
  3. 結果をスコア降順にマージし、テキストが重複するものを除去
  4. 上位 k 件を返却
出力: [{"text": str, "source": str, "page": int, "score": float, "shared": bool}, ...]
```

- ChromaDB の distance は L2 距離。`score = 1 / (1 + distance)` で正規化する
- `shared: True` は共有コレクションのチャンクであることを示す

#### `has_any_sources(username: str) -> bool`

```
処理: 個人コレクション・共有コレクションいずれかに1件以上チャンクがあれば True を返す
用途: チャット画面でドキュメント未登録ガードに使用
```

#### `list_sources(username: str) -> list[str]`

```
処理: 指定コレクションのメタデータを全取得し source の重複排除リストを返す
出力: ["file_a.pdf", "file_b.csv", ...]
※ @st.cache_data(ttl=30) でキャッシュ
```

#### `list_sources_with_count(username: str) -> list[dict]`

```
出力: [{"source": str, "chunks": int}, ...]
※ @st.cache_data(ttl=30) でキャッシュ
```

#### `delete_source(source: str, username: str) -> None`

```
処理:
  1. collection.get(where={"source": source}) で対象チャンクの ID を取得
  2. collection.delete(ids=ids) で削除
  3. キャッシュをクリア
```

## rag/moderation.py

### 責務
禁止ワード設定の永続化と、チャット送信前のメッセージチェックを担う。

### データ形式（data/moderation.json）

```json
{
  "enabled": true,
  "blocked_words": ["個人情報", "住所", "マイナンバー"]
}
```

### 関数詳細

#### `load_config() -> dict`

```
処理: data/moderation.json を読み込んで返す。ファイルが存在しない場合はデフォルト値を返す
デフォルト: {"enabled": True, "blocked_words": []}
```

#### `save_config(config: dict) -> None`

```
処理: filelock で排他制御しながら data/moderation.json に書き込む
```

#### `check_message(text: str) -> str | None`

```
入力: チャット送信テキスト
処理:
  1. load_config() で設定を取得
  2. enabled が False なら即座に None を返す
  3. blocked_words の各ワードに対して re.search（大文字小文字無視）を実行
  4. マッチしたワードを返す。マッチなしなら None を返す
```

## rag/chain.py

### 責務
検索・プロンプト組み立て・Claude API 呼び出しを一連のパイプラインとして実行する。

### 関数詳細

#### `answer(query: str, history: list[dict]) -> dict`

```
入力:
  query   : ユーザーの質問
  history : [{"role": "user"|"assistant", "content": str}, ...]
出力:
  {"answer": str, "sources": list[dict]}
```

**処理フロー**

```
1. embedder.search(query, k=SEARCH_K) で関連チャンクを取得
2. チャンクが0件の場合 → {"answer": "資料に該当情報がありません。", "sources": []} を返す
3. チャンクを連結して context 文字列を組み立てる
4. システムプロンプトを構築
5. history + 今回の query を messages に組み立て
6. anthropic.Anthropic().messages.create() を呼び出す
7. 回答テキストと sources を返却
```

**プロンプト詳細**

```python
SYSTEM_PROMPT = """あなたは社内情報アシスタントです。
以下の【参考資料】のみを根拠として質問に回答してください。
資料に情報がない場合は「資料に該当情報がありません」と答えてください。
回答は日本語で簡潔に記述してください。

【参考資料】
{context}
"""
```

**messages 組み立て**

```python
messages = [
    *history,                          # 過去の会話履歴
    {"role": "user", "content": query} # 今回の質問
]
```

**API 呼び出しパラメータ**

| パラメータ | 値 |
|-----------|-----|
| model | 環境変数 `CLAUDE_MODEL`（デフォルト: `claude-sonnet-4-6`） |
| max_tokens | 1024 |
| system | 上記 SYSTEM_PROMPT |
| messages | 上記 messages |

## ui/documents.py

### 責務
ドキュメント管理 UI を描画する。管理者は個人用 / 全体共有のアップロード先を選択できる。

### 処理フロー

```
1. get_current_role() でログインユーザーの役職を取得
2. 管理者の場合:
   - ラジオボタンで「個人用」/ 「全体共有」を選択
   - 「全体共有」選択時は target_user = SHARED_USER ("__shared__") を使用
   - 一般ユーザーの場合: target_user = username
3. st.file_uploader() でファイルを受け取る（複数選択可）
4. アップロードボタン押下時:
   a. 既存ドキュメントと重複チェック → 上書き確認ダイアログを表示
   b. data/uploads/{target_user}/ にファイルを保存
   c. loader.load_document() でチャンク取得
   d. embedder.add_documents(chunks, target_user) で登録
   e. 完了メッセージを表示
5. embedder.list_sources_with_count(target_user) で登録済みファイル一覧を表示
6. 各ファイルに削除ボタンを配置
   a. embedder.delete_source(source, target_user) を呼び出す
   b. data/uploads/{target_user}/ からもファイルを削除する
```

## ui/chat.py

### 責務
メインエリアのチャット UI を描画し、禁止ワードチェックと回答生成を呼び出す。

### 処理フロー

```
1. embedder.has_any_sources(username) でドキュメント存在確認
   - なければ案内メッセージを表示して終了
2. session_state["messages"] から会話履歴を取得・表示
3. 各アシスタントメッセージの下に出典・フィードバックボタンを表示
4. st.chat_input() で質問を受け取る
5. 質問受信時:
   a. moderation.check_message(prompt) で禁止ワードチェック
      - 検出: 警告メッセージを表示して中断（メッセージ履歴には追加しない）
      - 未検出: 次ステップへ
   b. messages に追加して即座に表示
   c. chain.answer(prompt, history, username) を呼び出す（st.spinner 表示中）
   d. コスト累積（total_cost_usd / input_tokens / output_tokens）
   e. 回答と sources を session_state に保存・表示
6. 会話リセットボタンで session_state の全チャット関連データをクリア
```

**出典の表示形式**

```
▼ 出典を見る
  ┌────────────────────────────────┐
  │ [1] file_a.pdf (p.3)          │
  │ ...チャンクのテキスト...       │
  │                                │
  │ [2] file_b.csv (行 12)        │
  │ ...チャンクのテキスト...       │
  └────────────────────────────────┘
```

## エラー処理詳細

### loader.py

| 状況 | 処理 |
|------|------|
| 未対応拡張子 | `ValueError("Unsupported file type: {ext}")` を raise |
| PDF が読み取れない | `RuntimeError` を raise、UI 側でキャッチして表示 |
| CSV / Excel が空 | 空リストを返す（警告なし） |
| Word / Excel が破損 | `ValueError` を raise、UI 側でキャッチして表示 |

### chain.py

| 状況 | 処理 |
|------|------|
| API キー未設定 | `anthropic.AuthenticationError` をキャッチし固定メッセージを返す |
| API レートリミット | `anthropic.RateLimitError` をキャッチし「しばらくしてから再試行してください」を返す |
| その他 API エラー | 例外メッセージをそのまま返す |

## ui/admin.py

### 責務
管理者専用パネルを描画する。ユーザー管理・フィードバック統計・禁止ワード設定の3＋1タブ構成。

### 禁止ワード設定タブ（_render_moderation）処理フロー

```
1. moderation.load_config() で現在の設定を取得
2. st.toggle() でフィルターの有効/無効を切り替え（変更即時保存・rerun）
3. フォームで新規ワードを入力・追加
   - 空欄: エラー表示
   - 重複: 警告表示
   - 正常: blocked_words に追加して保存・rerun
4. 登録済みワード一覧を表示し、各行に削除ボタンを配置
```

## セキュリティ考慮事項

| 対象 | 対策 |
|------|------|
| ファイルアップロード | `data/uploads/{username}/` 固定。`os.path.basename()` でパストラバーサルを防止 |
| 共有コレクション | 書き込みは管理者のみ（role チェックは `get_current_role()` で実施） |
| 禁止ワード設定 | 管理者パネルのみからアクセス可能。`moderation.json` は filelock で排他制御 |
| API キー | `.env` を `.gitignore` に追加し、リポジトリに含めない |
| ファイルサイズ | Streamlit のデフォルト上限（200MB）に委ねる |
| 外部アクセス | Streamlit はデフォルトで localhost のみバインド。外部公開には追加設定が必要 |
