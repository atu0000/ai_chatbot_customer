# 03 詳細設計

## rag/loader.py

### 責務
アップロードされたファイルを読み込み、チャンク分割した辞書リストを返す。

### 関数詳細

#### `load_document(file_path: str) -> list[dict]`

```
入力: ファイルパス（文字列）
出力: [{"text": str, "source": str, "page": int}, ...]
```

**処理フロー**

```
1. 拡張子を判定 (.pdf / .csv / .txt / .docx / .xlsx)
2. 対応する読み込み関数に振り分け
3. テキストをチャンク分割
4. 各チャンクに source（ファイル名）・page を付与して返却
```

**拡張子別処理**

| 拡張子 | ライブラリ | ページ情報 |
|--------|-----------|-----------|
| .pdf | PyMuPDF (`fitz`) | ページ番号（1始まり） |
| .csv | pandas | 行番号（1始まり） |
| .txt | 標準 io | 固定値 1 |
| .docx | python-docx | 段落番号（1始まり） |
| .xlsx | openpyxl | シート名＋行番号 |

**チャンク分割アルゴリズム**

```python
def _split_text(text: str, chunk_size: int, overlap: int) -> list[str]:
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunks.append(text[start:end])
        start += chunk_size - overlap
    return chunks
```

- `chunk_size` / `overlap` は環境変数から読み込む（デフォルト: 500 / 50）
- 空文字チャンクは除外する

**CSV の扱い**

- 各行を `列名: 値, 列名: 値, ...` 形式の文字列に変換してからチャンク分割
- ヘッダー行は本文には含めず、各行変換時に列名として使用する

**Word (.docx) の扱い**

- `python-docx` で全段落のテキストを順に取得する
- 段落番号を page に記録する（見出し・本文を区別しない）
- 表（Table）内のセルも走査してテキストを抽出する

**Excel (.xlsx) の扱い**

- `openpyxl` で全シートを走査する
- 各行を `列A: 値, 列B: 値, ...` 形式の文字列に変換する（CSV と同様）
- page には `"シート名_行番号"` 形式の文字列を設定する
- 空行はスキップする

## rag/embedder.py

### 責務
ChromaDB の操作（追加・検索・削除・一覧）を一元管理する。

### 初期化

```python
import chromadb
from sentence_transformers import SentenceTransformer

CHROMA_PATH = "data/chroma"
COLLECTION_NAME = "documents"
EMBED_MODEL = "paraphrase-multilingual-mpnet-base-v2"

client = chromadb.PersistentClient(path=CHROMA_PATH)
collection = client.get_or_create_collection(COLLECTION_NAME)
model = SentenceTransformer(EMBED_MODEL)
```

- `PersistentClient` により再起動後もデータを保持する
- Embedding モデルはアプリ起動時に1度だけロードする

### 関数詳細

#### `add_documents(chunks: list[dict]) -> None`

```
入力: [{"text": str, "source": str, "page": int}, ...]
処理:
  1. 各チャンクの "text" を SentenceTransformer で Embedding に変換
  2. ID を "source_page_index" 形式で生成（重複登録防止）
  3. collection.upsert() でベクトル・本文・メタデータを保存
```

#### `search(query: str, k: int = 5) -> list[dict]`

```
入力: 質問文字列, 取得件数
処理:
  1. query を Embedding に変換
  2. collection.query() で上位 k 件を取得
  3. distance を score（0〜1、高いほど類似）に変換して返却
出力: [{"text": str, "source": str, "page": int, "score": float}, ...]
```

- ChromaDB の distance は L2 距離。`score = 1 / (1 + distance)` で正規化する

#### `list_sources() -> list[str]`

```
処理: collection.get() でメタデータを全取得し source の重複排除リストを返す
出力: ["file_a.pdf", "file_b.csv", ...]
```

#### `delete_source(source: str) -> None`

```
処理:
  1. collection.get(where={"source": source}) で対象チャンクの ID を取得
  2. collection.delete(ids=ids) で削除
```

## rag/chain.py

### 責務
検索・プロンプト組み立て・Claude API 呼び出しを一連のパイプラインとして実行する。

### 関数詳細

#### `answer(query: str, history: list[dict]) -> dict`

```
入力:
  query   : ユーザーの質問
  history : [{"role": "user"|"assistant", "content": str}, ...]
出力:
  {"answer": str, "sources": list[dict]}
```

**処理フロー**

```
1. embedder.search(query, k=SEARCH_K) で関連チャンクを取得
2. チャンクが0件の場合 → {"answer": "資料に該当情報がありません。", "sources": []} を返す
3. チャンクを連結して context 文字列を組み立てる
4. システムプロンプトを構築
5. history + 今回の query を messages に組み立て
6. anthropic.Anthropic().messages.create() を呼び出す
7. 回答テキストと sources を返却
```

**プロンプト詳細**

```python
SYSTEM_PROMPT = """あなたは社内情報アシスタントです。
以下の【参考資料】のみを根拠として質問に回答してください。
資料に情報がない場合は「資料に該当情報がありません」と答えてください。
回答は日本語で簡潔に記述してください。

【参考資料】
{context}
"""
```

**messages 組み立て**

```python
messages = [
    *history,                          # 過去の会話履歴
    {"role": "user", "content": query} # 今回の質問
]
```

**API 呼び出しパラメータ**

| パラメータ | 値 |
|-----------|-----|
| model | 環境変数 `CLAUDE_MODEL`（デフォルト: `claude-sonnet-4-6`） |
| max_tokens | 1024 |
| system | 上記 SYSTEM_PROMPT |
| messages | 上記 messages |

## ui/documents.py

### 責務
サイドバーのドキュメント管理 UI を描画する。

### 処理フロー

```
1. st.file_uploader() でファイルを受け取る（複数選択可）
2. アップロードボタン押下時:
   a. data/uploads/ にファイルを保存
   b. loader.load_document() でチャンク取得
   c. embedder.add_documents() で登録
   d. 完了メッセージを表示
3. embedder.list_sources() で登録済みファイル一覧を表示
4. 各ファイルに削除ボタンを配置
   a. 押下時に embedder.delete_source() を呼び出す
   b. data/uploads/ からもファイルを削除する
```

## ui/chat.py

### 責務
メインエリアのチャット UI を描画し、回答生成を呼び出す。

### 処理フロー

```
1. st.session_state["messages"] から会話履歴を取得・表示
2. 各アシスタントメッセージの下に出典を折りたたみ表示
3. st.chat_input() で質問を受け取る
4. 質問受信時:
   a. messages に追加して即座に表示
   b. chain.answer() を呼び出す（st.spinner 表示中）
   c. 回答と sources を session_state に保存
   d. 回答を表示、出典を expander で表示
5. 会話リセットボタンで session_state["messages"] を空にする
```

**出典の表示形式**

```
▼ 出典を見る
  ┌────────────────────────────────┐
  │ [1] file_a.pdf (p.3)          │
  │ ...チャンクのテキスト...       │
  │                                │
  │ [2] file_b.csv (行 12)        │
  │ ...チャンクのテキスト...       │
  └────────────────────────────────┘
```

## エラー処理詳細

### loader.py

| 状況 | 処理 |
|------|------|
| 未対応拡張子 | `ValueError("Unsupported file type: {ext}")` を raise |
| PDF が読み取れない | `RuntimeError` を raise、UI 側でキャッチして表示 |
| CSV / Excel が空 | 空リストを返す（警告なし） |
| Word / Excel が破損 | `ValueError` を raise、UI 側でキャッチして表示 |

### chain.py

| 状況 | 処理 |
|------|------|
| API キー未設定 | `anthropic.AuthenticationError` をキャッチし固定メッセージを返す |
| API レートリミット | `anthropic.RateLimitError` をキャッチし「しばらくしてから再試行してください」を返す |
| その他 API エラー | 例外メッセージをそのまま返す |

## セキュリティ考慮事項

- アップロードファイルの保存先を `data/uploads/` 固定とし、パストラバーサルを防ぐため `os.path.basename()` でファイル名を正規化する
- `.env` を `.gitignore` に追加し API キーをリポジトリに含めない
- ファイルサイズ上限は Streamlit のデフォルト（200MB）に委ねる
