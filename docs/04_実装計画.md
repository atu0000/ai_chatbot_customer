# 04 実装計画

## 実装方針

- フェーズを4段階に分け、各フェーズで動作確認できる状態を維持する
- 外部依存（Claude API / ChromaDB）はフェーズ2以降で導入し、フェーズ1は依存ゼロで完結させる
- 各ステップ完了後にスモークテスト（手動確認）を実施してから次へ進む

---

## フェーズ構成

| フェーズ | テーマ | 完了の定義 |
|---------|--------|-----------|
| Phase 1 | 環境構築とプロジェクト骨格 | `streamlit run app.py` が起動し UI の枠が表示される |
| Phase 2 | ドキュメント登録パイプライン | ファイルをアップロードしてChromaDBへの保存が確認できる |
| Phase 3 | 回答生成パイプライン | 質問を入力すると Claude からの回答と出典が表示される |
| Phase 4 | 仕上げ・品質向上 | エラー処理・UX 改善が完了し README が整備されている |

---

## Phase 1 — 環境構築とプロジェクト骨格

### ステップ 1-1: 仮想環境とディレクトリ作成

```bash
python -m venv .venv
.venv\Scripts\activate
mkdir rag ui data\uploads data\chroma
type nul > rag\__init__.py
type nul > ui\__init__.py
```

### ステップ 1-2: requirements.txt 作成とインストール

```
streamlit>=1.35
anthropic>=0.28
chromadb>=0.5
sentence-transformers>=3.0
pymupdf>=1.24
pandas>=2.0
python-docx>=1.1
openpyxl>=3.1
python-dotenv>=1.0
```

```bash
pip install -r requirements.txt
```

### ステップ 1-3: .env と .env.example 作成

```bash
copy .env.example .env
# .env に実際の ANTHROPIC_API_KEY を記入
```

### ステップ 1-4: app.py の骨格作成

- `st.set_page_config` と `load_dotenv` のみ記述
- サイドバー・メインエリアにプレースホルダーテキストを配置

### ステップ 1-5: .gitignore 作成

```
.venv/
.env
data/uploads/
data/chroma/
__pycache__/
*.pyc
```

**確認**: `streamlit run app.py` でブラウザに UI の枠が表示される

## Phase 2 — ドキュメント登録パイプライン

### ステップ 2-1: rag/loader.py 実装

実装順序:
1. `_split_text()` — チャンク分割（依存ゼロ、単体テスト可）
2. `_load_txt()` — TXT 読み込み
3. `_load_csv()` — CSV 読み込み（pandas）
4. `_load_pdf()` — PDF 読み込み（PyMuPDF）
5. `_load_docx()` — Word 読み込み（python-docx）
6. `_load_xlsx()` — Excel 読み込み（openpyxl）
7. `load_document()` — 拡張子判定と振り分け

**確認**: Python インタラクティブシェルで各形式のファイルを読み込み、チャンクリストが返ることを確認

### ステップ 2-2: rag/embedder.py 実装

実装順序:
1. モジュールレベルの初期化（ChromaDB クライアント・SentenceTransformer ロード）
2. `add_documents()`
3. `list_sources()`
4. `delete_source()`
5. `search()` は Phase 3 で実装

**確認**: `add_documents()` 実行後に ChromaDB の `data/chroma/` にファイルが生成されることを確認

### ステップ 2-3: ui/documents.py 実装

- `st.file_uploader()` でファイル受け取り
- アップロードボタン → `loader.load_document()` → `embedder.add_documents()` の呼び出し
- 登録済みファイル一覧の表示
- 削除ボタンの実装

### ステップ 2-4: app.py にサイドバーを組み込み

**確認**: ファイルをアップロードし、サイドバーに一覧が表示され、削除ができる

## Phase 3 — 回答生成パイプライン

### ステップ 3-1: rag/embedder.py に search() を追加

- `collection.query()` で類似チャンクを取得
- distance → score の変換

**確認**: `search("質問文")` を呼び出してチャンクリストが返ることを確認

### ステップ 3-2: rag/chain.py 実装

実装順序:
1. `_build_context()` — チャンクを context 文字列に整形
2. `answer()` — 検索 → プロンプト組み立て → Claude API 呼び出し → 返却

**確認**: `answer("質問", [])` を直接呼び出して回答辞書が返ることを確認

### ステップ 3-3: ui/chat.py 実装

- 会話履歴の表示
- `st.chat_input()` の接続
- `chain.answer()` の呼び出しと結果表示
- 出典の `st.expander` 表示
- 会話リセットボタン

### ステップ 3-4: app.py にチャット画面を組み込み

**確認**: エンドツーエンドで質問 → 回答 → 出典が表示される
