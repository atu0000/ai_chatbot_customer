# 01 要件定義

## プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | 業務向け RAG チャットボット |
| 目的 | 社内問い合わせ対応の自動化。Claude Code を活用した設計・改善を示すポートフォリオ成果物 |
| 想定利用者 | 社内スタッフ（非エンジニアを含む） |
| 実行環境 | Windows ローカル |

---

## ビジネス要件

### 解決したい課題
- 社内規程・マニュアル・FAQ 等への繰り返し問い合わせを削減する
- 情報が複数ファイルに散在しており、担当者が探す手間がかかる

### 達成基準
- アップロードしたドキュメントの内容に基づき、自然言語で回答できる
- 回答の根拠となる元テキスト（出典）を表示できる
- 非エンジニアが独力でドキュメントを追加・更新できる

---

## 機能要件

### F-01 ドキュメントアップロード
- PDF / CSV / TXT / Word (.docx) / Excel (.xlsx) ファイルを UI 上からアップロードできる
- 複数ファイルを一度にアップロードできる
- アップロード済みドキュメントの一覧を確認できる
- アップロード済みドキュメントを削除できる

### F-02 チャット UI
- テキスト入力欄からユーザーが質問を送信できる
- 質問・回答の会話履歴を画面上に表示する
- 会話履歴を1クリックでリセットできる
- 禁止ワードを含む送信はブロックし、該当ワードを通知する

### F-03 回答生成
- アップロードされたドキュメント（個人＋共有）をもとに LLM が回答を生成する
- 関連ドキュメントが存在しない場合は「資料に該当情報がありません」と返答する
- 直近の会話コンテキストを考慮した回答を生成する（マルチターン対応）

### F-04 回答根拠表示
- 回答生成に使用した元テキスト（チャンク）を出典として表示する
- 出典にはファイル名とページ番号（または行番号）を含める
- 出典テキストは展開/折りたたみで表示する

### F-05 共有ドキュメント管理（管理者専用）
- 管理者はドキュメント管理画面でアップロード先を「個人用」または「全体共有」から選択できる
- 全体共有にアップロードされたドキュメントはすべてのユーザーのチャットで参照される
- 共有ドキュメントの一覧確認・削除も管理者が行う

### F-06 禁止ワード設定（管理者専用）
- 管理者は管理者パネルから禁止ワードを追加・削除できる
- 禁止ワードフィルター全体の有効／無効をトグルで切り替えられる
- 禁止ワードはアプリ再起動後も永続化される

---

## 非機能要件

### 保守性
- コードの行数を最小限に抑え、1ファイル＝1責務の構成にする
- 外部ライブラリはメジャーかつドキュメントが充実したものに限定する
- 環境変数は `.env` ファイルで一元管理する

### パフォーマンス
- ドキュメントのベクトル化は初回アップロード時のみ実行し、再起動後も再利用する（永続化）
- 回答生成は30秒以内に完了することを目標とする

### セキュリティ
- API キーをソースコードにハードコーディングしない
- アップロードファイルはローカルの指定ディレクトリのみに保存する
- 管理者が設定した禁止ワードを含む送信はブロックし、情報漏洩・悪意のある入力を予防する
- アプリはデフォルトで localhost のみバインドし、外部から直接アクセスできない構成とする

### 移植性
- Python 標準の `venv` で依存関係を管理する
- `requirements.txt` で再現可能な環境を提供する

---

## 技術スタック

| レイヤー | 採用技術 | 選定理由 |
|----------|----------|----------|
| UI | Streamlit | Python のみで完結、学習コストが低い |
| LLM | Anthropic Claude API | 日本語精度が高く、ポートフォリオ訴求に合致 |
| Embedding | `sentence-transformers` (multilingual) | ローカル実行可、日本語対応 |
| ベクトル DB | ChromaDB | 永続化が容易、追加設定不要 |
| ドキュメント解析 | PyMuPDF (PDF)、pandas (CSV)、標準 io (TXT)、python-docx (Word)、openpyxl (Excel) | 軽量で実績がある |
| 環境管理 | python-dotenv | シンプルな `.env` 管理 |

---

## 制約条件

- ローカル実行のみ（サーバーデプロイ不要）
- Windows 11 で動作すること
- Python 3.10 以上
- インターネット接続が必要（Claude API 呼び出し）
- GPU 不要（CPU のみで動作）

---

## スコープ外（今回実装しない）

- クラウドストレージ連携
- 回答品質の自動評価
- 部門（人事部など）単位での禁止ワード管理権限（現在は管理者のみ）
- 外部公開・デプロイ対応

## 用語定義

| 用語 | 定義 |
|------|------|
| RAG | Retrieval-Augmented Generation。外部知識を検索してから回答生成する手法 |
| チャンク | ドキュメントを分割した小テキスト単位。ベクトル化の基本単位 |
| Embedding | テキストを数値ベクトルに変換したもの。意味的類似度の計算に使用 |
| 出典 | 回答生成に使用した元テキストとファイル情報 |
