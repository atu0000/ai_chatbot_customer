## データフロー

### ドキュメント登録フロー

```
1. ユーザーがファイルをアップロード
2. loader.py がファイル種別を判定し本文を抽出
3. テキストをチャンク（500文字、重複50文字）に分割
4. embedder.py が各チャンクを Embedding に変換
5. ChromaDB にベクトル＋メタデータ（ファイル名・ページ）を保存
6. UI にアップロード完了を表示
```

### 回答生成フロー

```
1. ユーザーが質問を入力・送信
2. 禁止ワードチェック（moderation.check_message）
   - 検出: 警告を表示して中断
   - 未検出: 次ステップへ
3. 質問を Embedding に変換
4. ChromaDB で個人コレクション・共有コレクションそれぞれを検索し、
   スコア降順にマージして上位 k 件を取得
5. チャンクと会話履歴をプロンプトに組み込む
6. Claude API に送信
7. 回答テキスト＋使用チャンク（出典）をUIに表示
```

### 共有ドキュメント登録フロー（管理者）

```
1. 管理者がドキュメント管理画面で「全体共有」を選択
2. ファイルをアップロード（または URL を入力）
3. embedder.add_documents(chunks, username="__shared__") を呼び出す
4. ChromaDB の documents___shared__ コレクションに保存
5. 以降、全ユーザーの回答生成時に参照される
```

## プロンプト設計（基本方針）

```
System:
  あなたは社内情報アシスタントです。
  以下の【参考資料】のみを根拠として質問に回答してください。
  資料に情報がない場合は「資料に該当情報がありません」と答えてください。
  回答は日本語で、簡潔に記述してください。

【参考資料】
{検索で取得したチャンクを列挙}

User:
{質問}
```

- 会話履歴は `messages` 配列に含め、マルチターンに対応
- ハルシネーション抑制のため「資料のみを根拠にする」を明示

## 状態管理（Streamlit Session State）

| キー | 型 | 内容 |
|------|----|------|
| `messages` | `list[dict]` | 会話履歴（role / content） |
| `sources_map` | `dict[int, list]` | メッセージインデックス→出典チャンクのマップ |
| `feedback_done` | `set[int]` | フィードバック済みメッセージインデックス |
| `total_cost_usd` | `float` | セッションの累計コスト（USD） |
| `total_input_tokens` | `int` | セッションの累計入力トークン数 |
| `total_output_tokens` | `int` | セッションの累計出力トークン数 |
| `uploader_key` | `int` | ファイルアップローダーのリセット用キー |
| `pending_overwrites` | `list` | 上書き確認待ちのファイルリスト |

## エラーハンドリング方針

| エラー種別 | 対処 |
|-----------|------|
| 未対応ファイル形式 | UI にエラーメッセージを表示、処理中断 |
| API キー未設定 | 起動時に `.env` を検証、未設定なら警告表示 |
| Claude API エラー | エラーメッセージをチャット欄に表示 |
| ドキュメント未登録で質問 | 「先にドキュメントをアップロードしてください」と案内 |
| 禁止ワード検出 | 送信をブロックし、検出ワードを警告表示 |
| Cookie キー未設定 | 起動時に検証し、デフォルト値・空欄なら即座にエラー停止 |
